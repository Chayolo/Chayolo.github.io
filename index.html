About SQL Inject

发表于 2023-04-22 更新于 2023-05-16

SQL注入漏洞产生的原理：

  • SQL 注入就是指 web 应用程序对用户输入的数据合法性没有过滤或者是判断，前端传入的参数是攻击者可以控制，并且参数带入数据库的查询，攻击者可以通过构造恶意的 sql 语句来实现对数据库的任意操作。

    对于上述原理的补充与理解：

    1：web应用为什么能向后台数据库传递SQL语句进行数据库操作(即为什么参数可以代入数据库的查询)？

    • Web应用程序是基于客户端/服务器模式开发的，其中客户端向服务器发送请求，服务器接收请求后进行处理并返回响应结果给客户端
    • Web应用通常采用的是一种称为“动态页面”的技术，即前端页面通过向后台服务器发起请求，后台服务器接收请求后生成动态页面并返回给前端展示。而生成动态页面的过程中，需要从数据库中读取数据，此时就需要使用SQL语句进行数据库操作
    • Web应用通过与后台数据库通信，将用户的请求转化成SQL语句并提交到后台数据库，后台数据库再根据SQL语句执行相应的数据库操作
